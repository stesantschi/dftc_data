# AI Use

Claude (Anthropic) Opus and Sonnet models were used as a tool in building this dataset across
2025–2026. 

Use: 
- scaffolding and debugging the harvest and reshaping scripts (NDL harvesting, toponym extraction); 
- rule-bound repetitive work (romanization, encoding checks, date inference, deduplication logic); 
- proposing and testing matching methods, including discarding ones that failed; 
- drafting this documentation based on in-LLM project notes.

Not delegated were the scholarly judgments, such as which works are georeferenceable, what to exclude, 
reading place names and assigning coordinates, artist attribution, and name standardization. 
AI suggestions were treated as drafts and checked against source authorities (institutional catalogues, NDL,
Nichibunken, Wikidata), then corrected or dropped. 
Automated matching was constrained where it produced confident wrong answers.

AI output can be wrong while sounding right, so the dataset is not presented as
finished: the curation log lists provisional entries, unverified attributions,
and known gaps, keeping the AI-assisted steps auditable.
